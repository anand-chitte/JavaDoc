Joins

Joins allow you to combine rows from two or more tables based on a related column. Different types of joins provide different ways to retrieve data.

1. INNER JOIN

Retrieves only matching rows from both tables.

Example:

SELECT employees.name, departments.department_name
FROM employees
INNER JOIN departments ON employees.department_id = departments.id;

2. LEFT JOIN (LEFT OUTER JOIN)

Retrieves all rows from the left table and matching rows from the right table. If no match is found, NULL is returned.

Example:

SELECT employees.name, departments.department_name
FROM employees
LEFT JOIN departments ON employees.department_id = departments.id;

3. RIGHT JOIN (RIGHT OUTER JOIN)

Retrieves all rows from the right table and matching rows from the left table. If no match is found, NULL is returned.

Example:

SELECT employees.name, departments.department_name
FROM employees
RIGHT JOIN departments ON employees.department_id = departments.id;

4. FULL JOIN (FULL OUTER JOIN)

Retrieves all rows from both tables. If there is no match, NULL values are used where no data is found.

Example:

SELECT employees.name, departments.department_name
FROM employees
FULL JOIN departments ON employees.department_id = departments.id;

5. CROSS JOIN

Returns the Cartesian product of both tables (i.e., every row from the first table is combined with every row from the second table).

Example:

SELECT employees.name, departments.department_name
FROM employees
CROSS JOIN departments;

6. SELF JOIN

A join where a table is joined with itself.

Example:

SELECT e1.name AS Employee, e2.name AS Manager
FROM employees e1
INNER JOIN employees e2 ON e1.manager_id = e2.id;

7. NATURAL JOIN

Automatically joins tables based on common column names.

Example:

SELECT employees.name, departments.department_name
FROM employees
NATURAL JOIN departments;

Notes:

  • Always use ON with JOINs (except for NATURAL JOIN).
  • INNER JOIN is the default join type if you just use JOIN.
  • CROSS JOIN should be used with caution as it may produce a large number of rows.
